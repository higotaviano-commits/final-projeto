# User Stories - Smart Clinic Management System

## 👨‍💼 Admin User Stories

### 1. Add Doctor
As an admin, I want to add new doctors to the system so that patients can schedule appointments with them.

### 2. View Doctors
As an admin, I want to view all registered doctors so that I can manage their information.

### 3. Delete Doctor
As an admin, I want to remove doctors from the system so that outdated records can be cleaned.

### 4. Manage System
As an admin, I want to manage users and system data so that the platform remains organized and secure.

---

## 👨‍⚕️ Doctor User Stories

### 1. View Appointments
As a doctor, I want to see all my scheduled appointments so that I can manage my daily schedule.

### 2. Manage Availability
As a doctor, I want to define my available times so that patients can book appointments accordingly.

### 3. View Patient Details
As a doctor, I want to access patient information so that I can provide better care.

### 4. Create Prescription
As a doctor, I want to create prescriptions for patients so that treatments can be documented.

---

## 🧑‍🤝‍🧑 Patient User Stories

### 1. Search Doctor
As a patient, I want to search for doctors by name or specialty so that I can find the right professional.

### 2. Book Appointment
As a patient, I want to schedule appointments with doctors so that I can receive medical consultation.

### 3. View Appointments
As a patient, I want to see my appointments so that I can track my consultations.

### 4. View Prescriptions
As a patient, I want to view my prescriptions so that I can follow medical instructions.

---

## 📌 Notes

- The system supports three roles: Admin, Doctor, and Patient.
- The backend is built using Spring Boot with REST APIs.
- The frontend uses HTML, CSS, and JavaScript.
- Data is stored using MySQL and MongoDB.
