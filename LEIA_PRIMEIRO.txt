PROBLEMA ORIGINAL:
  ❌ No dashboard admin, ao clicar "Add", nada era salvo
  ❌ GET /doctors retornava sempre []
  ❌ Parecer que POST não estava funcionando

CAUSA RAIZ:
  A entidade Doctor (e todas as outras) NÃO tinha getters, setters e construtores
  JPA precisa disso para:
    1. Desserializar JSON → Objeto Java
    2. Serializar Objeto Java → JSON
    3. Persistir no banco de dados

SOLUÇÃO:
  Adicionar Lombok:
    @Data                    (gera getters, setters, equals, hashCode, toString)
    @NoArgsConstructor      (construtor sem argumentos para JPA)
    @AllArgsConstructor     (construtor com todos os argumentos)

RESULTADO:
  ✅ POST /doctors agora salva dados
  ✅ GET /doctors agora retorna dados
  ✅ Frontend atualiza automaticamente
  ✅ Dados persistem no H2 Database
  ✅ Sistema 100% funcional

ARQUIVOS MODIFICADOS:

  • Appointment.java - Adicionado Lombok
